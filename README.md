# Agile Software Development Lifecycle

## Overview

Successful software projects require more than writing code—they require effective planning, collaboration, communication, and continuous improvement. This repository demonstrates my understanding of the Software Development Life Cycle (SDLC) through the application of Agile and Scrum methodologies. Throughout this project, I explored how software teams gather requirements, prioritize work, collaborate with stakeholders, and deliver software through an iterative development process.

This project was completed as part of Southern New Hampshire University's **CS-250: Software Development Lifecycle** course and reflects the knowledge and documentation practices used throughout the software development process.

---

## Why the Software Development Life Cycle Matters

The Software Development Life Cycle provides a structured framework for planning, developing, testing, deploying, and maintaining software. Following an established development process helps teams improve software quality, reduce project risks, communicate more effectively, and deliver solutions that meet customer needs.

Understanding the SDLC is essential for software developers, business analysts, IT analysts, project managers, and other technology professionals who contribute to successful software projects.

## Agile Methodology

Agile is an iterative approach to software development that focuses on collaboration, adaptability, and delivering value through small, incremental releases. Rather than attempting to complete an entire project before gathering feedback, Agile encourages teams to continuously evaluate progress, respond to changing requirements, and improve throughout development.

During this project, I gained experience understanding how Agile practices support communication, planning, and continuous improvement throughout the SDLC.

---

## Scrum Framework

Scrum is one of the most widely used Agile frameworks. It organizes development work into short iterations called sprints while clearly defining team roles, planning activities, and project deliverables.

Learning Scrum helped me better understand how development teams organize work, prioritize customer requirements, manage changing priorities, and deliver functional software in an organized and repeatable process.
